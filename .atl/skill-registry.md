# Skill Registry

Índice regenerable de skills. Regenerar con `./scripts/setup.sh` o `pwsh ./scripts/setup.ps1`.

| Skill | Path | Trigger | Inputs | Outputs |
| --- | --- | --- | --- | --- |
| `branch-pr` | `skills/branch-pr` | Preparar la entrega de un change en una rama y un PR coherentes. Usar cuando el trabajo ya fue aplicado y verificado, y se necesite resumir alcance, checks, artifacts afectados y criterio de aceptación para revisión. | - Diff final del repo - Evidencia de verify - Templates de `.github/PULL_REQUEST_TEMPLATE.md` | - Resumen para PR - Checklist de revisión |
| `issue-creation` | `skills/issue-creation` | Convertir un change o una parte de sus tasks en issues trazables para seguimiento. Usar cuando se necesite descomponer trabajo en tickets que referencien changes, docs/product y criterios de verificación. | - `changes/<change-name>/tasks.md` - `proposal.md`, `spec.md` o `design.md` - Templates de `.github/ISSUE_TEMPLATE/` | - Issue draft o issue creado |
| `sdd-apply` | `skills/sdd-apply` | Aplicar cambios en documentación, código o estructura siguiendo el change activo. Usar cuando proposal, spec, design y tasks ya definieron qué debe implementarse o alinearse dentro del repositorio. | - Artifacts del change activo - Convenciones del repo - Scripts de setup/validación | - Archivos modificados o creados - Referencias actualizadas |
| `sdd-archive` | `skills/sdd-archive` | Cerrar o archivar un change una vez verificado y fusionado. Usar cuando sea necesario documentar el estado final, referencias del PR y cualquier transición del change activo hacia un estado histórico. | - Estado final del change - Evidencia de verify - Referencias del PR o merge | - Resumen o marca de cierre del change |
| `sdd-design` | `skills/sdd-design` | Redactar o ajustar el diseño técnico de un change en changes/<change-name>/design.md. Usar cuando la spec requiera decisiones de arquitectura, modelo de datos, integración, persistencia o tradeoffs antes de ejecutar tareas. | - `spec.md` - `docs/architecture.md` - Restricciones técnicas del repo | - `changes/<change-name>/design.md` |
| `sdd-explore` | `skills/sdd-explore` | Explorar el contexto funcional y técnico existente antes de proponer cambios. Usar cuando sea necesario leer docs/product, changes vigentes y documentación marco para comprender alcance, restricciones e impacto. | - `docs/product/` - `changes/<change-name>/` - `docs/architecture.md` y `README.md` | - Resumen de contexto - Riesgos y dependencias - Referencias para el change activo |
| `sdd-init` | `skills/sdd-init` | Inicializar o alinear la estructura base de un repositorio orientado a specification-driven development. Usar cuando se necesite crear o reparar la capa estructural del repo, verificar la presencia de docs, changes, skills, scripts, .atl o templates de colaboración. | - Estado actual de la raíz del repo. - Contenido existente en `docs/product/` y `changes/`. - Reglas operativas definidas en `AGENTS.md`. | - `AGENTS.md` - `README.md` - `CONTRIBUTING.md` - Estructura base del framework |
| `sdd-propose` | `skills/sdd-propose` | Redactar o ajustar la propuesta de un change en changes/<change-name>/proposal.md. Usar cuando se necesite definir intención, problema, alcance, riesgos, rollback y criterios iniciales del cambio. | - Hallazgos de `docs/product/` - Contexto de `changes/` relacionados - Restricciones del repo y del PR | - `changes/<change-name>/proposal.md` |
| `sdd-spec` | `skills/sdd-spec` | Redactar o ajustar la especificación delta de un change en changes/<change-name>/spec.md. Usar cuando sea necesario formalizar requisitos ADDED, MODIFIED, REMOVED y escenarios de aceptación sobre la base de docs/product. | - `proposal.md` - Reglas de `docs/product/` - Changes relacionados en `changes/` | - `changes/<change-name>/spec.md` |
| `sdd-tasks` | `skills/sdd-tasks` | Desglosar un change en tareas accionables dentro de changes/<change-name>/tasks.md. Usar cuando se requiera convertir proposal/spec/design en un plan ejecutable y verificable por fases. | - `proposal.md` - `spec.md` - `design.md` - Convención compartida de tareas | - `changes/<change-name>/tasks.md` |
| `sdd-verify` | `skills/sdd-verify` | Verificar que un change y la estructura del repositorio cumplan criterios explícitos. Usar cuando se deban ejecutar checks, completar verify-checklist, validar skills, scripts y compatibilidad con docs/product y changes existentes. | - Cambio aplicado - `scripts/validate-structure.sh` - `verify-checklist.md` o `verify.md` | - Evidencia de validación - `verify-checklist.md` actualizado si aplica |
| `skill-registry` | `skills/skill-registry` | Regenerar y mantener el índice de skills del repositorio en .atl/skill-registry.md. Usar cuando se agreguen, renombren o ajusten skills, triggers, inputs u outputs y se necesite sincronizar el catálogo operativo. | - Carpetas bajo `skills/` - Frontmatter y cuerpo de cada `SKILL.md` - Scripts de setup | - `.atl/skill-registry.md` |
