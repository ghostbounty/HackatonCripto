# Ejemplo para Codex

## Cómo consumir el repo

1. Leer `AGENTS.md` para conocer flujo, triggers y rutas obligatorias.
2. Consultar `skills/` para elegir la fase del trabajo.
3. Revisar `docs/product/` antes de cualquier cambio funcional.
4. Crear o actualizar artifacts dentro de `changes/<change-name>/`.
5. Usar `.atl/skill-registry.md` como índice rápido de skills.
