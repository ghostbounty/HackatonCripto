# Task Stub Format

## Plantilla sugerida

```md
# Tasks: <change-name>

## 1. Discovery
- [ ] Revisar `docs/product/`
- [ ] Revisar artifacts del change

## 2. Definition
- [ ] Confirmar proposal/spec/design

## 3. Execution
- [ ] Aplicar cambios
- [ ] Regenerar artifacts auxiliares

## 4. Verification
- [ ] Ejecutar validaciones
- [ ] Actualizar checklist de verify
```

## Reglas

- Usar casillas accionables.
- Mantener agrupación por fase.
- Referir rutas concretas del repo cuando el trabajo dependa de ellas.
