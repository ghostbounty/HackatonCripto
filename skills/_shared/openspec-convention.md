# OpenSpec Convention

## Objetivo

Definir la convención documental mínima para cambios trazables sin alterar el contexto funcional existente.

## Convención

- Ubicar cada cambio en `changes/<change-name>/`.
- Mantener como artifacts base `proposal.md`, `spec.md`, `design.md`, `tasks.md` y `verify-checklist.md` o `verify.md`.
- Usar `docs/product/` como fuente funcional y de negocio.
- Usar `.atl/skill-registry.md` como índice regenerable de skills.

## Orden sugerido

1. Proposal
2. Spec
3. Design
4. Tasks
5. Apply
6. Verify
7. Archive
